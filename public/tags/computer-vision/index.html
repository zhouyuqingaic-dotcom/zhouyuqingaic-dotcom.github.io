论文基本信息 Title: 5%>100%: Breaking Performance Shackles of Full Fine-Tuning on Visual Recognition Tasks arXiv: 2408.08345 Venue: IEEE/CVF Conference on Computer Vision and Pattern Recognition (CVPR) / arXiv 2024 1.核心目标 当构建极其依赖视觉“证据（Evidence）”与大模型“上下文（Context）”进行协同推理的网络架构时，视觉端的特征提取是整个系统成败的阿喀琉斯之踵。 这里的“两难困境”，本质上是“领域知识的极度渴求”与“预训练泛化先验的保护”之间的严重物理冲突。具体来说，分为以下两端： 困境一：全量微调带来的“流形坍塌”与“资源黑洞 为了让网络具备极高的视觉精度（比如能精准识别医疗影像中低对比度的毛玻璃影、极其微小的病灶边缘），最直觉的做法是放开整个视觉编码器（Vision Encoder），用特定领域的图像进行 100% 全量微调。但这条路会踩中两个致命陷阱： 视觉流形的灾难性遗忘： 视觉大模型（如高分辨率的 ViT 或 Swin）在海量自然图像上预训练时，建立了一个极度宽广、鲁棒的“通用视觉流形”。如果用数据量相对较小、且视觉特征极其单一（如只有灰阶的医学 X 光片）的垂直域数据去全量冲洗它，模型会迅速过拟合。它原本能识别万物的底层感受野会发生空间坍塌。这就导致当它进入后续能力对齐阶段（如 Stage 2）与 LLM 协同推理复杂场景时，视觉端提供的特征已经变得畸形且狭隘。 算力与显存的灾难： 如果频繁对高维度的多模态大模型底座进行全量参数更新、甚至涉及庞大的梯度检查点保存，通信开销和显存占用都是难以承受的，极大地限制了算法迭代和多阶段（Stage 0 -> Stage 1 -> Stage 2）实验的吞吐效率。 困境二：冻结骨干网络带来的“空间拓扑撕裂”与“证据模糊” 为了规避上述的灾难，常规的退路是：彻底冻结（Freeze）视觉主干网络，完全保留其泛化先验，仅仅在特征输出端接一个线性投影层（MLP Projector / Q-Former），或者使用标准的 NLP 线性微调方案（如 QLoRA）。但这对于需要“精准语义注入”的架构来说，无异于饮鸩止渴： ...
